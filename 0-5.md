# Bandit Levels 0-5

## Level 2

The filename was starting with `-`

* To read such files use `./`
* Syntax:

```bash
cat ./-
```

or

```bash
cat < ./*
```

### Learned

* Files starting with `-` can be interpreted as command options.
* Using `./` tells Linux to use the file from the current directory.

---

## Level 3

The filename had spaces.

Example:

```text
--spaces in this filename--
```

* To read files with spaces use quotes.

Syntax:

```bash
cat "--spaces in this filename--"
```

### Learned

* Quotes help Linux treat the whole filename as a single argument.

---

## Level 4

Used command:

```bash
reset
```

* When terminal looks messy, weird, or unreadable.
* Resets the terminal.

### Learned

* `reset` is different from `clear`.
* `clear` only clears the screen.
* `reset` reinitializes the terminal.

---

## Level 5

Finding files by size.

Syntax:

```bash
find . -size 100c
```

Sizes:

```bash
find . -size 100c
```

100 bytes

```bash
find . -size 1k
```

1 KB

```bash
find . -size 1M
```

1 MB

Useful modifiers:

```bash
find . -size +100c
```

Greater than 100 bytes

```bash
find . -size -100c
```

Less than 100 bytes

Checking file size:

```bash
ls -l filename
```

Finding non-executable files:

```bash
find . ! -executable
```

### Learned

* Using `find` with size filters.
* Identifying executable and non-executable files.
* Checking file size using `ls -l`.

---

## Level 6

Hidden files.

Used:

```bash
ls -a
```

### Learned

* Hidden files start with `.`
* `ls -a` shows hidden files.
